# Scripts & Makefile Review — 2026-07-19

This document records the issues found during a review of `Makefile`,
`scripts/sync-prod.sh`, and `scripts/push-to-prod.sh`, plus the fixes that were
applied. It exists so that future Claude (or any agent) sessions can audit
what changed and why.

Source files reviewed:
- `Makefile`
- `scripts/sync-prod.sh`
- `scripts/push-to-prod.sh`

## Severity legend

- **CRITICAL** — risk of data loss, silent failure, or production damage
- **HIGH** — correctness bugs that produce wrong output in normal use
- **MEDIUM** — UX / consistency / operational friction
- **LOW** — nice-to-have polish

## Behavioural changes summary

For users running `make sync-prod` / `make push-prod`:

1. `make test` is faster and cleaner (no `.venv` in coverage).
2. `make clean` removes `htmlcov/`.
3. `make lint` now shellchecks `scripts/*.sh` if `shellcheck` is installed.
4. `make migrations` no longer hangs on EOF.
5. `make sync-prod` and `make push-prod`:
   - Are stricter about what they accept (bad flags, bad jobs values).
   - Surface real errors instead of silently reporting success.
   - Use `PGPASSWORD` env var instead of inlining passwords in connection strings.
   - Keep the local DB out of a half-restored state if `pg_restore` fails.
6. `push-to-prod.sh` gains:
   - `--yes` to skip the confirm prompt in CI.
   - `--exclude-tables` (with `refresh_tokens` default-excluded).
   - Schema pre-check that aborts before pushing if local has a column prod doesn't.
   - Timezone-aware comparisons.
   - Cleanup trap for intermediate dumps on failure.
